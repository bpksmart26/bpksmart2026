# STAGE 1 — 지금 가능한 작업

## ① 노션 Integration 만들기 (2분)

- [ ] https://www.notion.so/profile/integrations 접속
- [ ] **+ New integration** 클릭
- [ ] 설정값:
  - 이름: `BPK Smart Sync` (자유)
  - Type: **Internal**
  - Workspace: 본인 워크스페이스 선택
- [ ] **Capabilities** 3개 모두 체크
  - [x] Read content
  - [x] Update content
  - [x] Insert content
- [ ] Submit 후 표시되는 **Internal Integration Secret** 복사
  - 형식: `secret_...` 또는 `ntn_...`
  - ⚠️ **노션 페이지·공개 채팅·git에 절대 붙여넣지 말 것**

---

## ② 노션 데이터베이스 만들기 (15~25분)

- [ ] 노션에서 새 페이지 만들기
- [ ] **+ 누르고 Database — Full page** 선택
- [ ] DB 제목: `BPK 통합 고객 관리` (자유)
- [ ] 기본 `이름` Title 컬럼을 **`회사명`으로 rename** ⭐
- [ ] `docs/notion-sync/노션DB속성목록.md`의 **39개 속성 전부 추가**
- [ ] 페이지 우측 상단 `⋯` 메뉴 → **Connections → BPK Smart Sync 연결** ⭐
  - ⚠️ 이걸 안 하면 모든 노션 호출이 404 실패
- [ ] 페이지 URL에서 **DB ID** 복사
  - URL 형태: `https://www.notion.so/{workspace}/{DB_ID}?v=...`
  - `?v=` 앞 32자 hex (영숫자) 부분이 DB ID

---

## ③ GAS Script Properties 등록 (1분)

- [ ] Apps Script 에디터 열기 (구글 시트 → 확장 프로그램 → Apps Script)
- [ ] 좌측 톱니바퀴 ⚙️ **Project Settings** 클릭
- [ ] 하단 **Script Properties** 섹션
- [ ] **Add script property** 두 개 추가:

| Property | Value |
|---|---|
| `NOTION_TOKEN` | ①에서 복사한 secret |
| `NOTION_DB_ID` | ②에서 복사한 32자 hex |

# STAGE 3 — 코드 적용 후에만 가능

> Claude가 GAS 코드(`syncFromNotion` 등)를 작성·붙여넣은 다음에 진행하세요.
> 코드 없는 상태에서 ④ 트리거 함수 드롭다운에 `syncFromNotion`이 안 보이는 것은 정상입니다.

## ④ [STAGE 3] 5분 폴링 트리거 등록 (1분)

- [ ] Apps Script 에디터 좌측 ⏰ **Triggers** 클릭
- [ ] 우측 하단 **+ Add Trigger**
- [ ] 다음 값으로 설정:

| 항목 | 값 |
|---|---|
| Function | `syncFromNotion` |
| Deployment | Head |
| Event source | Time-driven |
| Type | Minutes timer |
| Interval | Every 5 minutes |
| Failure notification | Daily |

---

## ⑤ [STAGE 3] GAS 새 버전 배포 (코드 적용 후, 코드 수정할 때마다)

- [ ] Apps Script 에디터 우측 상단 **Deploy → Manage deployments**
- [ ] 기존 배포 옆 ✏️ (편집 아이콘) 클릭
- [ ] **Version**: New version 선택
- [ ] Description: `노션 동기화 도입` 같은 메모
- [ ] Deploy 버튼 → 새 URL 발급
- [ ] 발급된 새 URL을 `config.js`의 `APPS_SCRIPT_URL`에 반영
- [ ] git commit + push

> ⚠️ 코드 수정 후 "왜 안 되지?" 의심되면 90%는 이 단계를 빼먹은 것

---

## ⑥ 노션 DB 페이지에 운영자 안내 callout 작성 (선택, 5~10분)

DB 상단에 운영 룰 안내 callout 추가. 다음 텍스트를 그대로 사용:

```
🔒 회색 컬럼은 시트가 source — 여기서 수정해도 다음 sync 때 되돌아갑니다.
   본 데이터(회사명·연락처·견적 항목 등)는 견적 웹페이지에서 수정하세요.

✏️ 양방향 컬럼: 신청상태 / 담당자 / 신청메모 / 견적메모
   이 4개는 노션에서 수정한 값이 시트로 반영됩니다.

❌ 노션에서 페이지 자체를 삭제하지 마세요.
   다음 동기화 때 새로 생성되며, 그 페이지에 적은 메모/하위페이지가 사라집니다.
   삭제는 견적 웹페이지의 "중복 정리" 모달에서 하세요.
```

- [ ] 노션 DB 페이지 상단에 callout 추가 (`/callout` 입력)
- [ ] 위 안내문 붙여넣기
- [ ] 색상은 노란색 또는 빨간색 추천 (눈에 띄게)
